# 小组开发协作规范
## 1. 分支命名规则
1. 新功能开发：feature/功能名称
例：feature/login、feature/homepage
2. Bug修复：fix/问题描述
例：fix/login-error
3. 文档修改：docs/文档内容
例：docs/update-readme

## 2. Commit提交注释规范
格式：`[#Issue编号] 简短描述改动内容`
示例：
[#1] 完成登录页面布局
[#docs] 更新README运行步骤

禁止模糊注释：修复bug、改代码、更新文件

## 3. PR合并流程
1. 完成开发后推送自己的功能分支；
2. 发起Pull Request，关联对应Issue；
3. 至少1名组员完成代码Review；
4. 评审通过无问题，再合并至main分支；
5. 合并完成后删除临时功能分支。

## 4. 代码验收标准
1. 代码能正常运行，无报错；
2. 代码缩进、命名统一；
3. 配套注释清晰，关键逻辑加说明；
4. 同步更新对应文档。# test2
